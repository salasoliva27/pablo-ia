  <files>dashboard/bridge/permissions.ts, dashboard/bridge/claude-session.ts</files>
  <action>
**permissions.ts** — Create a `PermissionManager` class:

- `pendingPermissions: Map<string, PendingPermission>` — stores pending permission promises keyed by toolUseID
- `TIMEOUT = 60_000` — 60 second timeout constant
- `request(ws: WebSocket, toolName: string, input: Record<string, unknown>, toolUseID: string): Promise<PermissionResult>` — sends `permission_request` ServerMessage over WS, creates a Promise stored in the map, sets a timeout that auto-denies after 60s
- `resolve(id: string, allowed: boolean): void` — clears timeout, removes from map, resolves the promise with `{ behavior: "allow" }` or `{ behavior: "deny", message: "User denied" }`
- `rejectAll(): void` — iterates all pending, clears timeouts, resolves with deny, clears map. Called on WS disconnect.

**claude-session.ts** — Create a `ClaudeSession` class:

- Constructor takes `ws: WebSocket` and `permissionManager: PermissionManager`
- `activeQuery: ReturnType<typeof query> | null` — tracks the active SDK query
- `async start(prompt: string, cwd: string): Promise<void>`:
  1. Call `query({ prompt, options: { cwd, permissionMode: "default", includePartialMessages: true, persistSession: true, canUseTool: (toolName, input, opts) => this.permissionManager.request(this.ws, toolName, input as Record<string, unknown>, opts.toolUseID) } })`
  2. Store as `this.activeQuery`
  3. Iterate with `for await (const message of this.activeQuery)`: send each as `{ type: "claude_message", message }` via WS
  4. On result message (check `message.type === "result"`), also send `{ type: "session_end", cost: message.total_cost_usd, usage: message.usage }`
  5. Wrap in try/catch — on error send `{ type: "error", message: err.message }`
- `async followUp(prompt: string): Promise<void>`:
  1. If no activeQuery, send error message over WS and return
  2. Call `activeQuery.streamInput()` with an async generator yielding a single user message: `{ type: "user" as const, session_id: "", message: { role: "user", content: prompt }, parent_tool_use_id: null }`
- `async interrupt(): Promise<void>`: if activeQuery, call `activeQuery.interrupt()`
- `close(): void`: if activeQuery, call `activeQuery.close()`, set to null

Import `query` from `@anthropic-ai/claude-agent-sdk`. Import types from `./types.js`. Import `PermissionManager` from `./permissions.js`. Import `WebSocket` from `ws`.

Do NOT use `--bare` flag — Venture OS needs MCP servers and hooks to function as the master orchestrator (per research anti-patterns).
