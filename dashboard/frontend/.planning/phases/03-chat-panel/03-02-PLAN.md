Create `ToolCallCard.tsx` (per D-04):
- Props: `{ toolName: string; input: Record<string, unknown>; result?: string }`
- Uses native `<details>/<summary>` for collapse (no library)
- Summary shows tool name with a wrench/tool icon (unicode or SVG)
- Body shows `JSON.stringify(input, null, 2)` in a `<pre>` with mono font
- If result exists, show below input separated by a divider
- Styling: border var(--border-color), border-radius var(--border-radius-md), margin var(--space-2) 0, summary has cursor pointer and padding var(--space-2) var(--space-3)

Create `PermissionPrompt.tsx` (per D-05):
- Props: `{ id: string; toolName: string; input: Record<string, unknown>; status: "pending" | "approved" | "denied" | "timeout"; onRespond: (id: string, allowed: boolean) => void }`
- Shows tool name and a brief description of what it wants to do
- Two buttons: "Approve" (green, var(--color-success)) and "Deny" (red, var(--color-danger))
- Buttons disabled when status !== "pending"
- After click, show status text ("Approved" / "Denied") instead of buttons
- Timeout countdown: if status is "pending", show seconds remaining (start from 60, decrement via setInterval, clear on unmount or status change)
- Styling: distinct from regular messages — border-left 3px solid var(--color-accent), background var(--color-bg-surface)
