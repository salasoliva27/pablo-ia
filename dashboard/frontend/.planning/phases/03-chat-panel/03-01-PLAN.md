Create `src/types/chat.ts` with discriminated union types:
```typescript
export type ChatMessage = UserMessage | AssistantMessage | ToolCallMessage | PermissionMessage;

export interface UserMessage {
  type: "user";
  id: string;
  content: string;
  timestamp: number;
}

export interface AssistantMessage {
  type: "assistant";
  id: string;
  content: string; // accumulated markdown text
  timestamp: number;
  isStreaming: boolean;
}

export interface ToolCallMessage {
  type: "tool_call";
  id: string;
  toolName: string;
  input: Record<string, unknown>;
  result?: string;
  timestamp: number;
}

export interface PermissionMessage {
  type: "permission";
  id: string; // matches permission_request id
  toolName: string;
  input: Record<string, unknown>;
  status: "pending" | "approved" | "denied" | "timeout";
  timestamp: number;
}
```

Create `src/components/ChatPanel.tsx`:
- Props: `{ lastMessage: ServerMessage | null; send: (msg: ClientMessage) => void; status: ConnectionStatus }`
- Use `useReducer` for messages state with actions: ADD_USER_MESSAGE, APPEND_ASSISTANT_TEXT, ADD_TOOL_CALL, ADD_PERMISSION_REQUEST, RESOLVE_PERMISSION, SET_STREAMING_DONE
- `useEffect` watches `lastMessage` and dispatches appropriate actions:
  - `claude_message`: parse `message.message.content` array. For `text` blocks, append to current assistant message (or create new one). For `tool_use` blocks, add ToolCallMessage. Check `message.type` — if "assistant", it may contain partial content (streaming).
  - `permission_request`: add PermissionMessage with status "pending"
  - `session_end`: mark last assistant message as not streaming
  - `error`: add as assistant message with error styling
- Render messages list in a scrollable div with ref
- Auto-scroll logic: track `userScrolledUp` via onScroll. If scrolled within 50px of bottom, auto-scroll on new messages. Use `useEffect` with messages.length dependency.
- Pass `onSend` callback to ChatInput
- `onSend` dispatches ADD_USER_MESSAGE locally, then calls `send({ type: hasSession ? "follow_up" : "start", prompt })`. Track `hasSession` state (set true after first start, reset on session_end).
- Show connection status indicator at top of panel
